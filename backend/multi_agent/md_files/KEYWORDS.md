# Multi-Agent 시스템 키워드 가이드

각 에이전트가 처리하는 질문의 키워드 목록입니다.
Supervisor Agent는 이 키워드를 기반으로 적절한 에이전트를 선택합니다.

## 1. Chatbot Agent (일반 대화)

### 키워드
```
안녕, 하이, 헬로, 반가워, 고마워, 감사
날씨, 오늘, 내일, 어제, 시간, 몇 시
기분, 좋아, 싫어, 행복
잘 지내, 어떻게 지내, 뭐해, 뭐하니
놀자, 재미, 즐거워
좋은 하루, 좋은 밤, 잘 자, 안녕히
뭐야, 그게 뭐야
재밌어, 웃겨, 하하, 헤헤
```

### 예시 질문
- "안녕하세요!"
- "오늘 날씨 좋네요"
- "고마워요"
- "잘 지내?"

## 에이전트 선택 우선순위

### 1. 감정 키워드 우선
감정 표현(힘들어, 우울해, 스트레스 등)이 있으면 **Therapy Agent** 우선

```
"일이 너무 힘들어" → Therapy Agent
"스트레스 받아" → Therapy Agent
```

### 2. 회사/업무 정보 요청
규정, 정책, 문서 관련은 **RAG Agent**

```
"연차 규정" → RAG Agent
"복지 정책" → RAG Agent
```

### 3. 아이디어/전략 요청
창의적, 혁신, 아이디어 관련은 **Brainstorming Agent**

```
"새로운 아이디어" → Brainstorming Agent
"문제 해결 방법" → Brainstorming Agent
```

### 4. 일정/계획 관련
할 일, 일정, 계획은 **Planner Agent**

```
"오늘 할 일" → Planner Agent
"일정 정리" → Planner Agent
```

### 5. 실적/분석 요청
리포트, 실적, 분석은 **Report Agent**

```
"주간 리포트" → Report Agent
"실적 분석" → Report Agent
```

### 6. 일반 대화
위 카테고리에 해당하지 않는 일상 대화는 **Chatbot Agent**

```
"안녕하세요" → Chatbot Agent
"고마워" → Chatbot Agent
```

## 키워드 중복 처리

여러 에이전트의 키워드가 겹치는 경우:

### 예시 1: "오늘"
- Chatbot: "오늘 날씨 어때?" → 일반 대화
- Planner: "오늘 할 일" → 일정 관리
- Report: "오늘 실적" → 리포트

→ **문맥과 함께 나오는 다른 키워드로 판단**

### 예시 2: "기분"
- Chatbot: "기분 좋아" → 일반 대화
- Therapy: "기분이 우울해" → 심리 상담

→ **감정의 강도와 부정적 표현 여부로 판단**

### 예시 3: "시간"
- Chatbot: "지금 몇 시야?" → 일반 대화
- Planner: "시간 관리 방법" → 일정 관리

→ **질문의 목적(정보 vs 조언)으로 판단**

## 키워드 업데이트 방법

새로운 키워드를 추가하려면:

1. `backend/multi_agent/supervisor.py`의 `_create_supervisor_prompt()` 수정
2. 해당 에이전트의 키워드 섹션에 추가
3. 이 문서(`KEYWORDS.md`) 업데이트
4. 테스트 케이스 추가
